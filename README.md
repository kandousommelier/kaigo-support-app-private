# 伴走支援ナビ

伴走支援ナビは、介護事業所向けに「今やること」「今後の予定」「現在の進捗」「提出・回答リンク」「資料・動画」「成果物」「FAQ」「用語集」「利用上の注意・情報管理」「問い合わせ」を確認できるGitHub Pages向けの静的Webアプリです。

本アプリは、案内と導線を提供するためのページです。回答データ、個人情報、個別ケースの詳細、施設内部の機密情報は保持しません。実データや提出物は、Googleフォーム、Googleドライブ、その他の外部リンク先で管理してください。

## 公開方法

ビルドは不要です。以下のファイルをリポジトリの直下に置いた状態でGitHub Pagesを有効化すれば、そのまま公開できます。

```text
index.html
assets/
data/
docs/
README.md
.nojekyll
```

`index.html`、`assets/`、`data/` は相対パスでつながっているため、リポジトリ名が `banso-support-navi` でも `accompaniment-support-navi` でも動作します。`.nojekyll` はGitHub Pages側で余計な変換を行わせないためのファイルです。

ローカルで確認する場合は、JSONファイルを読み込むために簡易サーバー上で開いてください。GitHub Pages上ではそのまま動作します。ファイルを直接ダブルクリックして開いた場合、ブラウザの制限でJSONが読み込めないことがあります。

## GitHub Pagesでの公開手順

1. GitHubで新しいリポジトリを作成します。リポジトリ名は `banso-support-navi` または `accompaniment-support-navi` を想定しています。
2. `index.html`、`assets/`、`data/`、`docs/`、`README.md`、`.nojekyll` をリポジトリの直下に配置します。
3. 変更を `main` ブランチへ反映します。
4. GitHubのリポジトリ画面で `Settings` を開きます。
5. 左側メニューの `Pages` を開きます。
6. `Build and deployment` の `Source` で `Deploy from a branch` を選びます。
7. `Branch` で `main`、フォルダで `/root` を選び、`Save` を押します。
8. 数十秒から数分待つと、同じ `Pages` 画面に公開URLが表示されます。
9. 表示されたURLを開き、伴走支援ナビの入室画面が表示されることを確認します。
10. 公開URLを `docs/施設向け案内文.md` の `伴走支援ナビURL` に記載し、施設へ案内します。

公開後に内容を直したい場合は、該当するJSONファイルを編集して `main` ブランチへ反映してください。GitHub Pages側に反映されるまで、少し時間がかかることがあります。

## 公開後の初回テスト手順

GitHub Pagesの公開URLが表示されたら、施設へ案内する前に以下を確認してください。

1. 公開URLを開き、入室画面に「伴走支援ナビ」と表示されることを確認します。
2. 共通ID `banso`、共通パスワード `support2026`、施設コード `shiga01` を入力し、施設別画面が表示されることを確認します。
3. 施設別画面で「今やること」「今後の予定」「現在の進捗」「提出・回答リンク」「資料・動画」「成果物」「FAQ」「用語集」「利用上の注意・情報管理」「問い合わせ」が表示されることを確認します。
4. URL未設定の指定項目が「メールにてお知らせ」または「近日公開予定」と表示され、リンクボタンとして開けないことを確認します。
5. 施設コードを `nagano01` や `kawasaki01` に変えて入室し、施設名や自治体名が切り替わることを確認します。
6. 非表示のリザーブ施設コード `reserve01` では入室できず、「施設コードが確認できませんでした」と表示されることを確認します。
7. 誤ったIDまたはパスワードを入力し、「IDまたはパスワードが一致しません」と表示されることを確認します。
8. スマホで公開URLを開き、カードが1列で読みやすく、ボタンが押しやすいことを確認します。
9. 入室画面と施設別画面に、個人情報を入力しない注意が表示されることを確認します。
10. 問い合わせセクションの「問い合わせフォームはこちら」から、問い合わせフォームが開けることを確認します。

## 入室情報

初期値は `data/config.json` で管理しています。

```text
共通ID: banso
共通パスワード: support2026
```

これは本格的な認証ではありません。関係者向けページであることと、情報管理に注意する必要があることを認識してもらうための入室作法です。公開されても支障のない情報だけを掲載してください。

## マスターデータの編集方法

表示内容は `/data` 配下のJSONファイルで管理します。アプリ本体を修正しなくても、JSONを編集すれば施設名、提出・回答リンク、FAQ、成果物などを更新できます。

```text
data/config.json          共通ID、共通パスワード、基本注意文
data/municipalities.json  自治体マスター
data/facilities.json      施設名、施設コード、サービス種別、現在ステップ、今やること
data/steps.json           ステップマスター
data/pages.json           画面セクション
data/schedules.json       今後の予定、参加者、予定補足
data/progress.json        現在の進捗、各ステップの状態と説明
data/links.json           提出・回答リンク、よく使うリンク、AI活用リンク
data/contents.json        資料・動画
data/deliverables.json    成果物名、目的、使用タイミング、テンプレートURL、記入例URL
data/faq.json             FAQのカテゴリ、質問、回答
data/terms.json           用語集
data/notices.json         利用上の注意
data/contact.json         問い合わせフォームURL、担当、電話、対応時間
```

JSON編集後は、カンマの付け忘れや余分なカンマがないか確認してください。

よくある更新は以下です。

- 施設名を変える: `data/facilities.json` の `facilityName` を編集します。
- 施設ごとの「今やること」を変える: `data/facilities.json` の `nextAction` を編集します。
- 今後の予定の日付や参加者を変える: `data/schedules.json` の `events` を編集します。
- 現在の進捗や「今ここ」を変える: `data/progress.json` の `items` を編集します。
- 提出リンクを追加・変更する: `data/links.json` の `url`、`title`、`description` を編集します。
- URL未確定のリンクに案内文を表示する: `data/links.json` の `url` を空文字にし、`displayText` に「メールにてお知らせ」などの表示文言を設定します。
- FAQを追加・変更する: `data/faq.json` を編集します。
- 成果物テンプレートを追加・変更する: `data/deliverables.json` の `templateUrl` や `sampleUrl` を編集します。
- 資料や動画を追加・変更する: `data/contents.json` を編集します。
- 問い合わせフォームや電話番号を変える: `data/contact.json` の `formUrl`、`phone`、`hours` を編集します。

## 施設追加方法

1. `data/facilities.json` に施設を1件追加します。
2. `facilityCode` は半角英数字で一意にします。
3. `municipalityCode` は `data/municipalities.json` のコードに合わせます。
4. 施設に表示したい現在ステップを `currentStep` に設定します。
5. 公開する施設は `isVisible: true` にします。

リザーブ10施設は初期状態で `isVisible: false` です。利用を開始する場合は、施設名やサービス種別を設定し、`isVisible` を `true` に変更してください。

## FAQ追加方法

`data/faq.json` に項目を追加します。

```json
{
  "faqId": "faq-example",
  "category": "伴走支援全体",
  "question": "質問文",
  "answer": "回答文",
  "municipalityCode": "common",
  "facilityCode": "common",
  "sortOrder": 120
}
```

特定自治体だけに表示したい場合は `municipalityCode` を `shiga` などに変更します。特定施設だけに表示したい場合は `facilityCode` に施設コードを設定します。

## リンク追加方法

`data/links.json` に項目を追加します。

```json
{
  "linkId": "common-new-link",
  "municipalityCode": "common",
  "facilityCode": "common",
  "category": "提出・回答",
  "title": "リンク名",
  "description": "リンクの説明",
  "url": "https://example.com/",
  "status": "利用可",
  "note": "注意事項",
  "sortOrder": 100
}
```

URLが未確定の場合は `url` を空文字にしてください。画面に表示したい案内文がある場合は、`displayText` に「メールにてお知らせ」や「近日公開予定」などを設定します。

表示優先順位は、施設別、自治体別、共通の順です。

## 載せてよい情報

- 伴走支援の一般的な案内
- 提出・回答フォームへのリンク
- 資料・動画へのリンク
- 成果物テンプレートへのリンク
- FAQ
- 用語集
- 情報管理上の注意
- 公開されても支障のないスケジュールや作業案内

## 載せてはいけない情報

- 利用者名、職員名、家族名など個人が特定される情報
- 個別ケースの詳細
- インタビュー全文
- 施設内部の生々しい課題
- 行政共有前の所感
- 回答データ
- 機密情報や公開されると支障のある情報

## 注意事項

伴走支援ナビは、GitHub Pagesで公開される前提の静的Webアプリです。共通ID、共通パスワード、施設コードはクライアント側で簡易チェックされます。認証システムではないため、アプリ内には公開されても支障のない情報だけを掲載してください。

ID・パスワード入力は、関係者向けページであることを認識してもらうための入室作法です。個人情報、施設固有の生々しい情報、回答データ、行政共有前の所感は伴走支援ナビ内に保持しないでください。
